Lista de Exercícios: Erros e Exceções em Python

Esta lista foi projetada para consolidar os conhecimentos acumulados de lógica, coleções e orientação a objetos, integrando-os com técnicas robustas de tratamento de erros (try/except) e criação de exceções customizadas.

Super Fácil

1. Protegendo a Divisão por Zero

Crie a função dividir_numeros(a, b). Ela deve tentar realizar a divisão de a por b. Caso o usuário tente passar o valor zero no segundo parâmetro, capture o erro de divisão por zero nativo do Python e retorne a mensagem textual: "Erro: Não é possível dividir por zero."

Super Fácil

2. Conversão Segura de Tipos

Crie a função converter_para_inteiro(texto). A lógica deve tentar converter a string recebida em um número inteiro e retornar esse número. Se a string contiver letras ou caracteres inválidos para conversão, capture o erro de valor e retorne textualmente: "Conversão inválida."

Super Fácil

3. Minha Primeira Exceção Customizada

Crie uma classe de erro customizada chamada ErroDeEntradaInvalida que herda da classe base de exceções do Python. Não adicione lógica interna nela (use apenas a palavra-chave de omissão).

Super Fácil

4. Disparando Erros Manuais

Crie a função verificar_sinal(numero). Se o número recebido for menor do que zero, use a palavra-chave adequada para disparar a exceção nativa ValueError com o texto explicativo: "O número não pode ser negativo."

Fácil

5. Acessando Índices de Listas com Segurança

Crie a função obter_elemento(lista, indice). Ela deve tentar acessar e retornar o valor localizado no índice informado. Se o índice não existir dentro da lista, capture a exceção de índice fora do limite e retorne a mensagem: "Posição inexistente."

Fácil

6. Condicionais Combinadas com Exceções

Crie a função validar_usuario(nome). Se o nome recebido for uma string vazia ou conter menos de 3 caracteres, force o disparo de um ValueError. Caso contrário, retorne textualmente: "Usuário válido."

Fácil

7. Capturando Múltiplos Erros Individuais

Crie a função processar_dados(lista, indice). O código deve tentar pegar o elemento da lista naquela posição e dividi-lo por 2. Monte um bloco que capture especificamente o erro se o índice não existir, e outro bloco separado se o elemento encontrado não for um número válido para operação matemática.

Fácil

8. Lógica de Erro em Construtores de Classes

Crie uma classe chamada Produto cujo construtor recebe nome e preco. Se o preço informado for menor ou igual a zero, o construtor deve disparar imediatamente um ValueError impedindo que o objeto chegue a ser instanciado.

Fácil

9. Usando o Bloco Finally

Crie a função simular_banco_de_dados(comando). Ela deve tentar rodar um print com o comando. Caso ocorra qualquer erro genérico no processo, capture-o. Independentemente de dar erro ou funcionar com sucesso, use a estrutura adequada para exibir sempre no final a mensagem: "Conexão encerrada."

Fácil

10. Exceção Customizada de Idade

Crie uma classe de exceção chamada IdadeInvalidaError. Depois, crie a função cadastrar_eleitor(idade). Se a idade for menor que 16, dispare o seu erro customizado.

Médio

11. O Som Polimórfico Protegido

Crie uma classe mãe Instrumento com um método tocar(self) que dispara um AnatomiaError (uma exceção customizada que você deve criar) avisando que classes abstratas não tocam som. Crie uma classe filha Guitarra que sobrescreve o método retornando um som clássico. Teste chamar o método das duas classes usando tratamento de erro.

Médio

12. Buscando Chaves por Índice de Texto

Crie a função buscar_letra_na_lista(lista_de_strings, indice_lista, indice_palavra). Ela deve acessar a lista no primeiro índice e buscar a letra da palavra usando o segundo índice. Trate as duas falhas possíveis (índice da lista inexistente ou palavra curta demais para o índice da letra) usando um único bloco de captura genérico, mas exibindo a mensagem nativa do erro capturado na tela.

Médio

13. Cadastro de Alunos com Notas Seguras

Crie a função adicionar_nota_aluno(lista_notas, nova_nota). A função deve verificar através de condicionais se a nota está entre 0.0 e 10.0. Se não estiver, lança um ValueError. Se o tipo do dado passado não for float ou int, capture a exceção de tipo adequada e exiba um aviso de tipo incorreto.

Médio

14. Elevador e Limite de Carga

Crie uma classe chamada Elevador que inicia com o atributo interno peso_total = 0. Crie o método entrar_pessoa(self, peso_pessoa). Se a soma do peso ultrapassar 400, dispare uma exceção criada por você chamada ElevadorSobrecargadoError e não altere o peso atual do elevador.

Médio

15. Calculadora Científica Experimental

Crie a função calcular_raiz_quadrada(numero). Caso o número seja negativo, use condicionais para disparar uma exceção chamada NumeroNegativoError. Caso passem um caractere de texto, capture o erro de tipo clássico. Junte tudo sob uma estrutura try/except externa.

Hard

16. O Banco e a Exceção de Saldo Insuficiente

Crie uma classe customizada chamada SaldoInsuficienteError que guarda internamente quanto saldo faltou para uma operação. Crie uma classe ContaBancaria com saldo inicial. O método sacar(self, valor) deve verificar se há dinheiro disponível. Se não houver, dispare o seu erro customizado calculando a diferença em dinheiro que faltou para o saque ser efetuado.

Hard

17. Validando Herança de Atributos Críticos

Crie uma classe mãe chamada Usuario e as classes filhas Admin e Comum. Crie a função deletar_banco_de_dados(usuario_objeto). A função deve testar se o objeto passado é da classe Admin usando funções de verificação de classe do Python. Se for um usuário Comum, force o disparo de uma exceção do tipo PermissionError com um texto explicativo de segurança.

Hard

18. Sistema de Login com Tentativas Limitadas

Crie uma classe chamada Autenticador que possui os atributos internos senha_correta = "1234" e tentativas = 0. Crie o método fazer_login(self, senha_digitada). A cada erro de senha, adicione 1 ao contador de tentativas e dispare um ValueError. Se o contador chegar a 3, qualquer tentativa nova de login deve disparar uma exceção customizada chamada ContaBloqueadaError.

🔥 Desafio Lendário

19. Simulador de Inventário de RPG com Polimorfismo Restrito

Crie uma classe mãe Item e as classes filhas Espada, Pocao e PedraComum. Crie uma classe chamada Inventario que possui uma lista interna vazia. O método guardar_item(self, objeto_item) deve adicionar o item na lista. Porém, se o objeto recebido for uma instância da classe PedraComum, o método deve recusar o armazenamento disparando uma exceção customizada chamada ItemInutilException. Se o inventário já tiver mais de 3 itens de qualquer tipo, ele deve disparar um InventarioCheioException.

🔥 Desafio Lendário

20. Processador de Transações Financeiras em Lote

Crie a função processar_lote_saques(conta_objeto, lista_de_valores). Essa função recebe um objeto de conta bancária (da questão 16) e uma lista contendo diversos valores numéricos de saques misturados com algumas strings inválidas inseridas por erro humano de digitação. A função deve percorrer a lista inteira tentando efetuar os saques. Se encontrar uma string, deve capturar o erro, exibir um aviso e continuar o laço. Se disparar o erro de saldo insuficiente, deve interromper imediatamente o lote todo com um aviso crítico na tela e salvar uma lista com os saques que deixaram de ser executados por falta de fundos.
